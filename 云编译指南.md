# 网吧USB管理软件 - 云编译指南

## 概述
本指南提供在云端编译网吧USB管理软件的详细步骤。由于项目使用.NET Framework 4.7.2（依赖Windows专属组件），必须在Windows环境中编译。

## 云编译方案对比

### 方案一：GitHub Actions（推荐，免费）
- **费用**: 免费（公开仓库）或 GitHub Pro（私有仓库）
- **优点**: 集成度高，配置简单，支持Windows构建
- **限制**: 每月2000分钟构建时间（公开仓库无限）
- **适合**: 快速编译，自动构建

- **费用**: 免费套餐（5个用户，1800分钟/月）
- **优点**: 专业CI/CD，完整Windows支持
- **限制**: 需要注册Azure账号
- **适合**: 企业级部署

### 方案三：自定义云服务器（AWS/Azure/腾讯云）
- **费用**: 按需计费（约¥2-10元/次）
- **优点**: 完全控制，可反复使用
- **限制**: 需要配置环境
- **适合**: 需要定制编译环境

### 完整GitHub Actions配置

创建文件 `.github/workflows/build.yml`:

```yaml
name: .NET Framework 4.7.2 Build

on:
  push:
    branches: [ main ]
  pull_request:
    branches: [ main ]
  workflow_dispatch:  # 允许手动触发

jobs:
  build:
    runs-on: windows-latest  # 必须使用Windows环境
    
    steps:
    - uses: actions/checkout@v3
    
    - name: Setup .NET Framework
      uses: microsoft/setup-msbuild@v1.1
      
    - name: Setup NuGet
      uses: nuget/setup-nuget@v1
      
    - name: Restore NuGet packages
      run: nuget restore CyberCafeUsbManager.sln
      
    - name: Build with MSBuild
      run: |
        msbuild CyberCafeUsbManager.sln `
          /p:Configuration=Release `
          /p:Platform="Any CPU" `
          /p:DeployOnBuild=true `
          /p:PublishProfile=FolderProfile `
          /verbosity:minimal
      
    - name: Verify output files
      run: |
        dir "CyberCafeUsbManager\bin\Release\"
        echo "检查文件大小..."
        $exe = Get-Item "CyberCafeUsbManager\bin\Release\CyberCafeUsbManager.exe"
        if ($exe.Length -lt 50000) { exit 1 }
        echo "EXE文件大小: $($exe.Length) bytes"
      
    - name: Create deployment package
      run: |
        # 创建临时目录结构
        New-Item -ItemType Directory -Force -Path "deployment-package\主程序"
        New-Item -ItemType Directory -Force -Path "deployment-package\脚本工具"
        New-Item -ItemType Directory -Force -Path "deployment-package\配置文件"
        New-Item -ItemType Directory -Force -Path "deployment-package\说明文档"
        
        # 复制主程序文件
        Copy-Item "CyberCafeUsbManager\bin\Release\*" -Destination "deployment-package\主程序\" -Recurse
        
        # 复制配置文件
        Copy-Item "usb_device_database.json" -Destination "deployment-package\配置文件\" -ErrorAction SilentlyContinue
        Copy-Item "CyberCafeUsbManager\app.config" -Destination "deployment-package\配置文件\app.config.example" -ErrorAction SilentlyContinue
        
        # 复制脚本工具（如果存在）
        if (Test-Path "scripts") {
            Copy-Item "scripts\*.bat" -Destination "deployment-package\脚本工具\" -Recurse
        }
        
        # 复制说明文档（如果存在）
        if (Test-Path "部署包\说明文档") {
            Copy-Item "部署包\说明文档\*" -Destination "deployment-package\说明文档\" -Recurse
        }
        
        # 创建README
        @"
网吧USB管理软件 - 云编译部署包
编译时间: $(Get-Date -Format 'yyyy-MM-dd HH:mm:ss')
GitHub Actions运行ID: ${{ github.run_id }}
"@ | Out-File -FilePath "deployment-package\README.txt" -Encoding UTF8
        
        echo "部署包创建完成"
        dir "deployment-package" -Recurse
        
    - name: Upload build artifacts
      uses: actions/upload-artifact@v3
      with:
        name: CyberCafeUsbManager-Release
        path: |
          CyberCafeUsbManager/bin/Release/
          deployment-package/
        retention-days: 7
        
    - name: Create ZIP package
      run: |
        Compress-Archive -Path "deployment-package\*" -DestinationPath "网吧USB管理软件_部署包.zip" -Force
        echo "ZIP文件大小: $( (Get-Item '网吧USB管理软件_部署包.zip').Length ) bytes"
        
    - name: Upload ZIP artifact
      uses: actions/upload-artifact@v3
      with:
        name: 网吧USB管理软件_完整部署包
        path: 网吧USB管理软件_部署包.zip
        retention-days: 30
```

## 方案二：Azure DevOps 云编译

### 步骤一：创建Azure DevOps项目
1. 访问 https://dev.azure.com
2. 注册免费账号
3. 创建新项目: `CyberCafeUsbManager`

### 步骤二：配置构建流水线
1. 进入项目 → Pipelines → New pipeline
2. 选择"Azure Repos Git"或连接GitHub
3. 选择"Starter pipeline"
4. 替换为以下配置:

```yaml
trigger:
- main

pool:
  vmImage: 'windows-latest'

variables:
  solution: '**/*.sln'
  buildPlatform: 'Any CPU'
  buildConfiguration: 'Release'

steps:
- task: NuGetToolInstaller@1

- task: NuGetCommand@2
  inputs:
    restoreSolution: '$(solution)'

- task: VSBuild@1
  inputs:
    solution: '$(solution)'
    msbuildArgs: '/p:DeployOnBuild=true /p:PublishProfile=FolderProfile'
    platform: '$(buildPlatform)'
    configuration: '$(buildConfiguration)'

- task: ArchiveFiles@2
  inputs:
    rootFolderOrFile: '$(Build.BinariesDirectory)'
    includeRootFolder: false
    archiveType: 'zip'
    archiveFile: '$(Build.ArtifactStagingDirectory)/CyberCafeUsbManager.zip'
    replaceExistingArchive: true

- task: PublishBuildArtifacts@1
  inputs:
    PathtoPublish: '$(Build.ArtifactStagingDirectory)'
    ArtifactName: 'drop'
    publishLocation: 'Container'
```

### 步骤三：下载构建结果
1. 构建完成后，在"Artifacts"中下载`drop`文件夹
2. 包含编译好的程序文件和ZIP包

## 常见问题解决

### Q1: GitHub Actions构建失败
**错误**: "无法找到.NET Framework 4.7.2"
**解决**: 
1. 在工作流中添加.NET Framework安装步骤
2. 使用Windows Server 2022镜像（包含.NET 4.8）

### Q2: NuGet包恢复失败
**错误**: "无法恢复包"
**解决**:
```yaml
- name: Add NuGet sources
  run: |
    nuget sources Add -Name "nuget.org" -Source "https://api.nuget.org/v3/index.json"
```

### Q3: 构建成功但程序无法运行
**解决**:
1. 确保所有依赖DLL文件存在
2. 检查.NET Framework 4.7.2运行时已安装
3. 在Windows 10/11环境中测试

### Q4: 私有仓库构建时间不足
**解决**:
1. 转为公开仓库（推荐，免费无限构建）
2. 升级到GitHub Pro
3. 使用Azure DevOps免费套餐
